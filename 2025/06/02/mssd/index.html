Application-specific computer systemmsd

What is an embedded system?（什么是嵌入式系统）

  1. Application-specific computer system（专用的计算机系统

  2. Built into a larger system(嵌入到更大的系统之中)

  3. Often with real-time computing constraints(通常具有实时计算约束)

Why add an embedded computer to a larger system?（为什么要用嵌入式？

  1. Better performance(更好的性能)

  2. More functions and features(更多功能特性)

  3. Lower cost (更低的消费)

  4. More dependability(更高的可靠性)

微处理器（MPU）和微控制器（MCU）

MPU: Typically defined as a single processor core (CPU) supports at least instruction fetching, decoding, and executing.(通常定义为单个cpu支持至少指令获取，解码和执行。)

A single MPU may have many CPUs.(单个mpu可能有多个cpu)

Normally for general-purpose computing, but needs to be supported with memories and I/Os（通常用于计算，而且需要io的支持和内存的支持。）

mcu：Typically has a MPU (typically a single core) with embedded peripherals e.g. memory blocks, Digital I/Os, Analogue I/Os, and other basic peripherals（通常具有mpu，并且嵌入了外围设备。

Embedded systems rely on both MCU hardware peripherals and software to get everything done on time

（嵌入式系统依赖于MCU硬件外设和软件内容，才能够按时完成所有任务。）

In most embedded systems, MCUs are chosen to be the best solution, since they offer（MCUs的优点）

• Low development and manufacturing cost（低的开发成本和制造成本。
• Easy porting and updating（容易移植和更新
• Light footprint（轻量级的构架
• Relatively low power consumption（相对低的功耗
• Satisfactory performance for low-end products（低端产品充足的性能

ARM processor families

Cortex-A series (Applications processors)：Support OS and high-performance applications e.g. smartphones, Smart TV, smart books

A系列（应用处理器）：支持操作系统和高性能应用，例如智能手机，电视或者书籍。

Cortex-R series (Real-time processors with high performance and high reliability)：Support real-time processing and mission-critical control e.g. automotive braking system, powertrains.

R系列（高性能可靠的实时处理器）：支持实时处理和任务关键控制，例如汽车制动，动力系统。

Cortex-M series (Microcontroller)：Cost-sensitive solutions for deterministic microcontroller applications e.g. smart sensors

M系列（微控制器）：针对确定性微控制器应用的低成本解决方案，例如智能传感器。

ARM Processors vs. ARM Architectures（ARM处理器和ARM架构）

ARM architecture(ARM架构)
• Describes the details of instruction set, programmer’s model, exception model, and memory map(描述指令集，程序员模型，异常模型和内容映射的详细信息)
• Documented in the Architecture Reference Manual（记录在架构参考手册之中）

ARM processor（ARM处理器）
• Developed using one of the ARM architectures（使用ARM结构之一开发）
• More implementation details, such as timing information（更多实现细节）
• Documented in processor’s Technical Reference Manual（记录在处理器的技术参考手册中）

Cortex-M4 Processor Memory Map（Cortex-M4 处理器内存映射）

Processor Registers（处理器寄存器）

R0 – R12: general purpose registers（通用寄存器）

Low registers (R0 – R7) can be accessed by any instruction（可以被任何指令访问）

High registers (R8 – R12) can only be accessed by some instructions（只能被某些指令访问）

R13: Stack Pointer (SP)（栈指针）

Records the current address of the stack（记录栈当前的地址）
Used for saving the context of a program while switching between tasks（用于在任务切换时保存程序上下文）

R14: Link Register (LR)（链接寄存器）

The LR is used to store the return address of a subroutine or a function call（LR用于·存储·1子程序或者·函数调用的返回地址）
The program counter (PC) will load the value from LR after a function is finished（函数执行完毕后，程序计数器会从LR加载值。）

xPSR: combined Program Status Register（组合程序状态寄存器）

Provides information about program execution and ALU flags（提供程序执行和ALU标志的信息）

Cortex-M4 Instruction Set（Cortex-M4 指令集）

Instruction Format（指令格式）

Labels（标签）：

Place marker, marking the memory address of the current instruction（放置标记，标记当前的指令内存地址）
Used by branch instructions to implement if-then or goto（被分支指令用于实现if-then or goto）
Must be unique（必须唯一）

Mnemonic（助记符）：

The name of the instruction（指令的名称）
Operation to be performed by processor core（处理器核心执行的操作）

Operands（操作数）：
Operands

Registers（寄存器）
Constants (called immediate values)（常量（称为立即值））

Number of operands varies（操作数的数量不同）

No operands（无操作数）: DSB
One operand（一个操作数）: BX LR
Two operands（两个操作数）: CMP R1, R2
Three operands（三个操作数）: ADD R1, R2, R3
Four operands（四个操作数）: MLA R1, R2, R3, R4

Normally（通常情况）

operand1 is the destination register, and operand2 and operand3 are source operands.

（operand1 是目标寄存器，operand2 和 operand3 是源操作数。）

operand2 is usually a register, and the first source operand

（operand2通常是寄存器，第一个源操作数。）

operand3 may be a register, an immediate number, a register shifted to a constant number of bits, or a register plus an offset (used for memory access)

（operand3可能是寄存器，立即数，寄存器左移常数位数或寄存器加偏移量（用于内存访问））

Comments（注释）

Everything after the semicolon (;) is a comment（分号（；）之后的内容都是注释。）

Explain programmers’ intentions or assumptions（用于解释意图或者假设）

ARM Assembly Language

Memory Access（内存访问）

STM32F401 which is based on the ARM Cortex-M4 core, supports only little-endian
memory format.

(STM32F401 基于 ARM Cortex-M4 内核，仅支持小端模式内存格式。)

In little-endian format, the least significant byte (LSB) is stored at the lowest memory
address, and the most significant byte (MSB) is stored at the highest memory address.

(在小端模式内存格式中，最低有效字节（LSB）存储在最低的内存地址，而最高有效字节（MSB）存储在最高的内存地址。)

Pre-index（前索引）

前索引是指在执行内存访问（加载或存储）之前，先计算目标地址，并将计算后的新地址更新到基址寄存器中，然后使用这个新地址进行内存访问。

Post-index（后索引）

后索引是指先使用基址寄存器中的当前值作为目标地址执行内存访问，然后在访问完成后，将基址寄存器更新为 Rn + offset。

Arithmetic: Addition（算数：加法）

64-bit addition with ADDS and ADC（64 位加法使用 ADDS 和 ADC）

Arithmetic: Subtraction（算数：减法）

96-bit subtraction with SUBS, SBCS and SBC（96 位减法使用 SUBS、SBCS 和 SBC）
